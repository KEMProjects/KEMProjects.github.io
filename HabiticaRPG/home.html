Habitica RPG Demo

This is a working demo of a RPG made with RPG Maker MV and using Node.JS to connect to the Habitica website to retrieve user information. The goal is to show how to use Habitica user information to incorporate in an RPG game. My idea was that I could create a playable game where instead of progressing by levelling up, instead new areas were opened based on the progress made in Habitica

What is Habitica?

Link to Habitica

Habitica is a free habit and productivity app that treats your real life like a game. Habitica can help you achieve your goals to become healthy and happy.

How it Works

Log-in

The RPG Maker MV engine takes care of starting the game. The next step is to log-in to Habitica. This is where I created custom Node.JS code to connect to Habitica using their open API. I used another library to create the log-in screen, and behind the scenes it connects and fetches the data. See KEM_HabiticaConnect.js and KEM_HabiticaConvert.js for the custom code I wrote.

Game Loads

The next step in the demo is that after log-in, the game generates the user and party based on the data obtained from Habitica. The first four party members are used for display and monster fights. The Habitica data is converted to RPG stats, so the health and level information is accurate to the database.
